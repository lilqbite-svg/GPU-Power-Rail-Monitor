# GPU Power Rail Monitor

**GPU Power Rail Monitor** — Windows Forms-приложение для мониторинга питания видеокарты и анализа контактной группы коннектора в реальном времени.

Проект построен вокруг прямого чтения телеметрии из драйверов NVIDIA:
- **NVAPI** — ядро напряжения, topology/rail-данные и часть внутренних GPU-сенсоров;
- **NVML** — board power, температура, загрузка, лимиты и энергия;
- без зависимости от **LibreHardwareMonitor**;
- без запуска `nvidia-smi` как внешнего процесса в стандартном сценарии.

Приложение ориентировано на Windows x64 и собирается под **.NET 10**.

---

## Что умеет приложение

- показывает несколько GPU в отдельном overview и по вкладкам;
- строит **историю** по напряжению, мощности и току;
- экспортирует журнал в **CSV** и **JSON**;
- копирует текущий лог в буфер обмена;
- работает в **tray-режиме** и может показывать уведомления при выходе параметров за пороги;
- поддерживает **автопрофили** для разных GPU и типов коннекторов;
- визуализирует контактную группу коннектора и честно помечает состояние данных:  
  **measured / estimated / nominal / unavailable**;
- умеет анализировать типовые схемы:
  - **12V-2x6 / 12VHPWR**;
  - **PCIe 6-pin**;
  - **PCIe 8-pin**;
  - **PCIe 6+2-pin**;
  - несколько multi-connector вариантов для старших плат;
- хранит настройки локально и не требует ручной правки конфигов в репозитории.

## Как устроены данные

### Источники телеметрии

**NVIDIA / NVAPI**
- core voltage (`GPU Core Voltage`);
- topology/rail power;
- внутренние данные по GPU, если драйвер их отдаёт;
- полезно для прямой работы с NVIDIA-картами без сторонних библиотек.

**NVIDIA / NVML**
- board power;
- power limit;
- GPU temperature;
- GPU utilization;
- memory utilization;
- total energy.

### Важное ограничение

На большинстве обычных плат приложение **не получает реальный per-pin ток каждого контакта** коннектора 12V-2x6 / 12VHPWR.  
В таком случае UI честно показывает **estimated**-режим на основе доступной телеметрии.

Если карта и драйвер действительно отдают более точные rail-данные, приложение использует их и помечает соответствующим образом.

## Требования

- **Windows 10/11 x64**
- **.NET 10 SDK** или runtime совместимой версии

Рекомендуется запускать приложение от имени администратора, если драйвер или система ограничивают доступ к части телеметрии.

## Цветовые и статусные состояния

Приложение использует несколько уровней статуса:

- **measured** — значение измерено напрямую;
- **estimated** — значение вычислено по доступной телеметрии;
- **nominal** — значение считается штатным / нормальным;
- **unavailable** — данных нет.

Пороги можно изменить в окне **Settings**.

---

## Поддерживаемые типы коннекторов

### 12V-2x6 / 12VHPWR
16-pin схема с:
- 12 силовыми контактами;
- 4 сигнальными контактами.

### PCIe 6-pin
Стандартный шестиконтактный разъём питания.

### PCIe 8-pin
Восьмиконтактный разъём питания.

### PCIe 6+2-pin
Схема с базовым 6-pin блоком и отдельной 2-pin частью.

> Геометрия коннектора в UI сделана так, чтобы быть визуально понятной и в то же время не перегружать экран лишними подписями.

## Уведомления

Tray-уведомления не должны спамить одинаковым сообщением каждый цикл опроса.  
Уведомление показывается как событие состояния, а не как повторяющийся шум.

Если приложение сворачивается в tray:
- оно остаётся запущенным;
- доступно через значок в панели задач;
- не теряет контекст истории и журналов.

## Если данных нет

Если приложение показывает `unavailable` или только estimated-режим, причины обычно такие:

1. драйвер не отдаёт соответствующую метрику;
2. карта не поддерживает нужный путь телеметрии;
3. доступ ограничен правами;
4. подключён не тот тип GPU или не тот профиль коннектора.

В таких случаях стоит:
- обновить драйвер;
- запустить приложение от администратора;
- проверить, что видеокарта действительно поддерживает NVAPI/NVML-метрики;
- уменьшить интервал опроса, если система очень чувствительна к частым вызовам.
